# ECG MI Detection — Pure 1D-CNN with LLM-Guided Dynamic Attention

A deep learning pipeline for Myocardial Infarction (MI) detection from 12-lead ECG signals. The model uses a fully convolutional architecture (no LSTM) augmented with patient-specific lead attention weights generated by a large language model (Claude) or a fast rule-based fallback.

---

## How It Works

Standard ECG classifiers treat all 12 leads equally. In clinical practice, a cardiologist immediately focuses on specific leads depending on the suspected MI territory — inferior MIs show up in II, III, aVF; anterior MIs in V1–V4; and so on.

This project encodes that clinical reasoning into the model:

1. A patient profile (age, gender, symptoms, risk factors, history) is sent to **Claude** (or processed by offline rules).
2. The LLM returns importance weights for each of the 12 leads (e.g. V1–V4 get high weights for an anterior STEMI patient).
3. These weights are injected into a **Squeeze-and-Excitation gate** inside the CNN, dynamically re-weighting feature channels at inference time without retraining.

## Architecture

```
Input (B, 12, 1000)
      │
   Stem Conv (k=15) + MaxPool → (B, 64, 500)
      │
   DilatedResBlock 1  dilation=1, stride=2 → (B, 128, 250)
      │
   DilatedResBlock 2  dilation=2, stride=2 → (B, 256, 125)
      │   └── SqueezeExcite + LLM projection  ← patient weights injected here
      │
   DilatedResBlock 3  dilation=4, stride=2 → (B, 256, 63)
      │
   DilatedResBlock 4  dilation=8, stride=2 → (B, 256, 32)
      │
   CBAM Temporal Attention → (B, 256, 32)
      │
   MultiScaleStatPool (mean + max + std + skew) → (B, 1024)
      │
   MLP Classifier (1024 → 512 → 128 → 2)
      │
   Output: Normal / MI
```

Each `DilatedResBlock` contains two dilated Conv1d layers, a Squeeze-Excitation module, dropout, and a strided shortcut connection. Block 2's SE gate accepts an additive signal from the LLM lead-weight projection, making it the sole point where clinical context enters the model.

**Parameter counts (approximate):**

| Model | Params |
|---|---|
| ECG_1DCNN_LLMAttention | ~860k |
| ECG_1DCNN_Baseline (no LLM gate) | ~847k |
| LLM SE projection overhead | ~13k (≈1.5%) |
| Old CNN-LSTM baseline | ~866k |

Training speed is ~3–4× faster than an equivalent CNN-LSTM because all operations are fully parallel (no sequential recurrence).

## Data

The pipeline expects two NumPy arrays in the project root:

| File | Shape | Description |
|---|---|---|
| `X_balanced_ecg.npy` | `(N, 1000, 12)` | Raw ECG signal — N samples, 1000 time steps, 12 leads |
| `Y_balanced_ecg.npy` | `(N,)` | Binary labels — 0 = Normal, 1 = MI |

The dataset is automatically split 60 / 20 / 20 (train / val / test) with stratification. Per-lead z-score normalisation is fitted on the training set only and applied to all splits.

A compatible public dataset is [PTB-XL](https://physionet.org/content/ptb-xl/1.0.3/) (PhysioNet). Preprocess it to the shape above before running.

## Usage

### Train both models and run full evaluation

```bash
python main.py
```

### Use Claude API for patient-specific lead weights at inference

```bash
python main.py --api-key sk-ant-api-key-here
```

Without an API key the pipeline falls back to the offline rule-based weight system automatically — no inference disruption.

### Skip training and load saved checkpoints

```bash
python main.py --skip-training
```

Expects `llm_model.pt` and `base_model.pt` in the project root (saved automatically after training).

## LLM Lead Weighting

### Claude API mode

The system prompt instructs Claude to act as a clinical ECG expert and return a JSON object with per-lead importance weights (0.0–1.0). A patient profile like:

```python
patient = {
    'age': 62,
    'gender': 'male',
    'symptoms': 'crushing chest pain, ST elevation V1-V4',
    'risk_factors': ['hypertension', 'diabetes', 'smoking'],
    'history': 'Previous MI, LAD territory'
}
```

yields weights such as `{"V1": 0.95, "V2": 0.95, "V3": 0.90, "V4": 0.88, ...}` (anterior STEMI pattern). These are softmax-sharpened and injected into the model.

### Rule-based offline mode

When no API key is provided, `rule_based_lead_weights()` in `llm_attention.py` applies keyword matching on symptoms and history to select from five predefined clinical territory profiles (inferior, anterior, lateral, posterior, septal), then applies demographic modifiers for age, gender, diabetes, and hypertension. Latency is under 1 ms.

## Module Reference

### `config.py`
Central place for all constants. Edit `EPOCHS`, `BATCH_SIZE`, `MAX_LR`, `PATIENCE` here rather than in individual files.

### `dataset.py`
`load_data(x_path, y_path)` — loads arrays, splits, normalises, builds DataLoaders, and returns a class-weighted `CrossEntropyLoss` with label smoothing.

### `model.py`
- `ECG_1DCNN_LLMAttention` — full model with LLM gate. Use `set_patient_weights(tensor)` before inference and `clear_patient_weights()` after.
- `ECG_1DCNN_Baseline` — identical architecture without LLM projection (ablation).

### `llm_attention.py`
- `query_llm_lead_weights(patient, api_key)` — calls the Anthropic API.
- `rule_based_lead_weights(patient)` — offline fallback.
- `weights_to_tensor(raw_weights, temperature)` — softmax-sharpens a raw weight dict into a tensor. Higher `temperature` → more peaked distribution.

### `train.py`
- `train_epoch` / `eval_epoch` — single-epoch helpers.
- `train_model(model, train_loader, val_loader, criterion, name, ...)` — full loop with OneCycleLR, gradient clipping, mixed-precision on CUDA, and early stopping on validation AUC-ROC.

### `evaluate.py`
- `find_optimal_threshold(labels, probs, metric)` — sweeps 161 thresholds, returns the one maximising F1 or Youden index.
- `compute_metrics(model, test_loader, val_loader, criterion, ...)` — prints a comparison table at both `t=0.50` and the optimal threshold.

### `inference.py`
`predict_patient(model, ecg_signal, patient, api_key, use_llm, temperature)` — complete single-patient pipeline. Accepts `(12, T)` or `(T, 12)` signal arrays.

### `visualize.py`
- `plot_training_curves` — loss, AUC, ROC side-by-side.
- `plot_confusion_matrices` — heatmaps with sensitivity / specificity annotations.
- `visualise_patient_prediction` — 12-lead ECG strips colour-coded by attention weight, lead-weight bar chart, MI risk gauge, and clinical summary panel.
